Git 101

Some common programming issues

  • We need to regularly back up our code
  • We need to keep "milestones" when our code actually works!
  • We need to keep a history of our codebase
  • We need to share our code with other developers
  • We need to experiment with our code without affecting the master copy

Distributed version control (Git!)

  • Every developer has their own copy of the code, and all copies are seen as equally "valid"
  • There may be a "central" repository, but this is only by convention. Developers can work anywhere with their local copies
  • DVCS encourages small, incremental changes, and makes merging code between copies easy.
  • DVCS makes experimentation easy and straightforward
