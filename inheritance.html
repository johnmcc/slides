Inheritance 101

The problem

"How do we model multiple classes when they share properties or behaviour?"

For example:

  • A sparrow can fly, but so can a crow, a seagull etc.
  • A car has wheels, and an engine that can start.

We could repeat the code in each class, but this would violate the DRY principle. (Don't repeat yourself!)

Let's take a look.

A better way

We can move the common functionality to a super class.

Let's look at how to do this.

(We can and should check that our tests still pass - this is one of the benefits of working in a TDD fashion)

Terminology

  • Superclass - the parent class for one or more classes
  • Subclass - the child class of a parent
  • Inheritance - when a child class takes on the behaviour of a parent class
  • Overriding - when a child class overrides the functionality in a parent class - let's look at this in more detail.

Super!

If we want a class to do something unique, but also perform its parent's functionality, we can call super() in Ruby.

Limitations

What if we want to add a vehicle that doesn't have an engine? Or a bird that doesn't fly, like a penguin?

We could:

  • Overwrite start_engine to just return 'I don't have an engine'
  • Remove start_engine from vehicle and put it back on Car and Motorbike
  • Add another layer of inheritance to have 'EnginedVehicles' and 'HumanPoweredVehicles'
